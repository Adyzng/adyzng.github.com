Windows驱动安装失败，Error=0x0000000D

发表于 2016-07-31   |   分类于 Windows   |     |   阅读量 次

It was failed with error when installing driver failed.
When check windows device setup logs C:\Windows\INF\setupapi.dev.log.

You can see following errors

  • Failed to call server side to delete driver package. Error = 0x0000000D
  • Failed to delete package from Driver Store
  • Failed to add the driver package: The data is invalid
  • UpdateDriverForPlugAndPlayDevices: error 13

It was caused by windows service DeviceInstall was not started.
This service should be set to Automatic Start.
On client environment it was set to Manually Start, that cause our driver install failed.

Resolution

  • sc start DeviceInstall
  • set Automatic Start of DeviceInstall

MS: Windows 8.1 fails to install device drivers
