WinHTTP

WinHTTP是微软开发的一套封装Http协议的C风格API，支持Http和Https.

Microsoft Windows HTTP Services (WinHTTP) provides developers with a server-supported, high-level interface to the HTTP/1.1 Internet protocol. WinHTTP is designed to be used primarily in server-based scenarios by server applications that communicate with HTTP servers.

使用WinHTTP下载文件过程如下：

1. 创建Session

不使用代理WINHTTP_NO_PROXY_NAME

2. 创建Connection

指定服务器URL的时候不能带Http/Https前缀

3. 创建Request

Https需要指定WINHTTP_FLAG_SECURE

4. 发送请求

如果是Https请求，且服务器端证书是不授信的或者CN无效，会连接失败，这时候需要设置WinHttpOption忽略指定错误SECURITY_FLAG_IGNORE_CERT_CN_INVALID和SECURITY_FLAG_IGNORE_UNKNOWN_CA

5. 等待服务器响应

如果Http或者Https协议不对，服务器会返回无效响应ERROR_WINHTTP_INVALID_SERVER_RESPONSE。此时可尝试另一种协议，重试前记得关闭之前创建的Request句柄。

6. 获取文件大小

在服务器返回的Http头里边有字段Content-Length，表示待下载文件大小，单位Byte

7. 接收数据

循环调用WinHttpQueryDataAvailable直到文件结束
